# Canonical weekly data — single source for all Innovien dashboards

One file, updated once per week, that every dashboard reads: **`weekly_data.json`**.

## The idea
- You refresh **one** file each week from the accurate Comtrak exports.
- This Vercel scoreboard reads it for the **Stretch Scorecard** tab. Any other dashboard
  (the local spread dashboard, future apps) reads the **same** file — no duplicate logic.
- Other tabs here (Q2 Goal Tracking, $1,250 Raffle) stay **live from Notion**.

## Where it lives / how dashboards read it
- **Master copy:** `Finance Optimization/weekly_data.json` (generated by `build_weekly_data.py`).
- **This app:** a copy sits at the repo root; the API reads it, and Vercel also serves it
  publicly at **`https://<your-app>.vercel.app/weekly_data.json`**.
- **Any dashboard** (local HTML, Sheets, another Vercel app) can `fetch()` that URL — or read
  the master file locally — so they all show the same numbers.

## Weekly update (one step)
1. Drop the week's Comtrak CSVs in the Weekly Spread Email folder and run your existing
   `build_dashboard_from_folder.py` (produces `dashboard_embedded_data.json`).
2. Run **`python build_weekly_data.py`** → regenerates `weekly_data.json`.
3. Fill the `scorecard` block's start-pipeline numbers (see below), copy the file into
   `innovien-scorecard/`, commit & push → Vercel redeploys and every dashboard updates.

## Schema
```jsonc
{
  "meta":   { "week_ending", "label", "quarter_label", "baseline_week_start",
              "quarter_start", "quarter_end", "generated", "source" },
  "company":{ "weekly_spread", "qtr_start_spread", "year_total" },
  "by_am":      [ { "name","office","weekly","qstart","cumulative" } ],
  "by_recruiter":[ ... ],
  "offices":[...], "divisions":[...], "clients":[...],

  // Stretch Scorecard tab. Each field: a number pins it; null = use live Notion.
  "scorecard": {
    "net_new_starts", "avg_start_spread",
    "pending_count", "pending_avg_spread", "pending_total_spread",
    "lockup_count", "lockup_spread",
    "dumpin_count", "dumpin_spread",
    "active_consultants", "redeployed", "available_bench",
    "forecast": [ { "weekStart","plannedIn","plannedOut" } ]   // optional; empty = compute from Notion
  }
}
```
